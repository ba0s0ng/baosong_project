二、软件功能与技术特点

1. 系统概述

工业互联网机床监测平台是一款面向工业生产环境的综合性机床监测与管理平台。该系统采用前后端分离架构，集设备监测、数据分析、报警管理、数字孪生和维护记录于一体，为工业制造企业提供全面的设备生命周期管理解决方案，帮助企业提高设备利用率、降低故障率、优化维护策略。

2. 主要功能模块

2.1 仪表盘模块
- 系统总体运行状态监控
- 设备数量、在线状态、报警数量等关键指标实时展示
- 设备状态分布图表可视化
- 关键参数趋势分析图表
- 最近报警信息列表
- 定时自动刷新数据

2.2 设备管理模块
- 设备信息的增、删、改、查功能
- 设备详细信息展示（名称、型号、序列号、安装日期等）
- 设备实时运行状态监控
- 设备实时参数展示（温度、振动、电流、转速等）
- 设备故障信息记录与查看

2.3 数字孪生模块
- 基于Three.js的3D场景构建与渲染
- 设备3D模型展示与交互
- 设备实时参数数据绑定与可视化
- 设备运行状态动态更新
- 设备选择与详细信息查看
- 响应式场景适配

2.4 报警管理模块
- 报警规则的设置与管理
- 报警记录实时展示与查询
- 报警级别分类（高、中、低优先级）
- 报警状态跟踪与处理记录
- 报警数据统计与分析
- 自动报警通知机制

2.5 数据分析模块
- 设备性能参数历史趋势分析
- 多参数相关性分析
- 设备效率与利用率统计
- 预测性维护数据分析
- 多设备数据对比分析
- 自定义时间范围数据查询
- 多种图表展示（折线图、柱状图、散点图等）

2.6 维护记录模块
- 设备维护计划管理
- 维护记录的创建、查询与更新
- 维护类型分类（预防性维护、故障维修、校准等）
- 维护人员、时间、成本等信息记录
- 维护满意度评价
- 维护记录统计与分析

3. 技术特点

3.1 前端技术
- 采用HTML5、CSS3、JavaScript构建响应式用户界面
- 使用Bootstrap框架实现现代化、自适应的界面设计
- 集成ECharts实现多样化的数据可视化图表
- 基于Three.js实现三维数字孪生场景
- 模块化设计，各功能模块独立封装，易于维护和扩展
- 支持模拟数据自动生成，便于系统演示和测试

3.2 后端技术
- 采用Python语言开发，基于FastAPI框架构建高性能API服务
- 支持异步处理，提高系统并发能力
- 实现CORS跨域资源共享，支持前后端分离架构
- 定时任务机制，自动生成设备运行数据
- 报警规则自动检测与报警触发

3.3 数据管理
- SQLite数据库存储，轻量高效
- 完整的数据模型设计，包含设备、数据、报警、规则、维护等核心实体
- 灵活的查询接口，支持多条件数据筛选
- 数据缓存机制，提高查询效率

3.4 系统集成
- RESTful API设计，便于与其他系统集成
- 支持MQTT协议，可与工业设备直接通信
- 模块化接口设计，支持功能扩展

三、软件结构

1. 系统架构
- 前端层：基于Web技术构建用户界面，负责数据展示和用户交互
- 后端层：提供API服务，处理业务逻辑，管理数据存储
- 数据层：负责数据持久化存储和查询

2. 目录结构
- frontend/：前端源代码目录
  - css/：样式文件
  - js/：JavaScript模块（app.js, dashboard.js, machines.js, digital-twin.js, alarms.js, analytics.js等）
  - index.html：主页面
- backend/：后端源代码目录
  - app/：应用代码（main.py, routes.py, db.py, schemas.py, mqtt_client.py等）
  - requirements.txt：依赖配置
- data/：数据库文件

四、软件用途

本软件适用于各类工业制造企业，特别是拥有大量机床设备的机械加工、汽车零部件、航空航天等行业。系统可帮助企业：
- 实时监控设备运行状态，及时发现异常
- 分析设备性能数据，优化生产效率
- 预测设备故障，实现预防性维护
- 规范化设备维护管理，延长设备使用寿命
- 降低设备故障率和维护成本
- 提升企业数字化管理水平
